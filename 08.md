# 08 - MYSQL BACKEND(08): Tối ưu hoá phân trang từ 7s còn 1s với Table có 10.000.000 dữ liệu

## Vấn đề với OFFSET
Khi sử dụng `LIMIT M OFFSET N` với `N` quá lớn, MySQL buộc phải quét qua `N + M` dòng, sau đó vứt bỏ `N` dòng đầu tiên. Ở những bảng hàng chục triệu bản ghi, điều này gây tốn CPU và I/O khổng lồ.

## Kỹ thuật tối ưu (Cursor-based Pagination / Keyset Pagination)
Thay vì dùng `OFFSET`, hãy sử dụng chỉ mục có sẵn (thường là khóa chính) để làm mốc cho lần truy vấn tiếp theo:
`SELECT * FROM table_name WHERE id > [last_seen_id] ORDER BY id ASC LIMIT 20;`
Kỹ thuật này tận dụng triệt để B-Tree Index, truy xuất trực tiếp vào Node cần thiết với thời gian gần như không đổi.
