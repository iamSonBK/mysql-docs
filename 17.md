# 17 - MYSQL BACKEND(17): Đảm bảo tính nhất quán (Consistency) khi ghi đồng thời dữ liệu vào Redis và MySQL

## Vấn đề Cache Invalidation
Khi dữ liệu lưu ở MySQL thay đổi, Redis (đóng vai trò cache) cần được cập nhật. Nếu ghi đồng thời không khéo léo sẽ dẫn đến việc Redis lưu trữ dữ liệu cũ (Stale Data).

## Chiến lược Cache-Aside và Double Delete
* **Xóa Cache, không cập nhật Cache trực tiếp:** Khi cập nhật MySQL thành công, tiến hành xóa key trong Redis.
* **Delayed Double Delete:** Để xử lý triệt để race condition khi có luồng đọc xen vào giữa lúc xóa cache và cập nhật DB, hệ thống có thể thực hiện xóa Cache lần 1 -> Cập nhật MySQL -> Xóa Cache lần 2 sau một khoảng trễ nhỏ (ví dụ 500ms).
